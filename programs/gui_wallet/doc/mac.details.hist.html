Problems and solutions during building DECENT project on MAC OS



Problems during generation of Makefile


1. crypto++ library not found. During compilation DECENT core. This should be added to the README.
    Solution: (This I fixed with ‘brew install cryptopp’)

2. Could NOT find OpenSSL. Again should be documented in the README of core DECENT
    A) brew install openssl
    B) export OPENSSL_ROOT_DIR=/usr/local/opt/openssl

3. ‘gmp library not found’
    A) brew install gmp


Problems during the compilation and linking


1. secp256k1-zkp/autogen.sh: line 3: autoreconf: command not found
    brew install automake

2. Libtool library used but 'LIBTOOL' is undefined
    brew install libtool

3. A lot of boost errors (for example ‘error: no type named 'fcontext_t' in namespace 'boost::context’)
    Using boost version 1.62
    problem resolved by compiling and using boost 1.57 version ???


Now the problems and the hints connected to building GUI wallet (again on MAC OS)



Problems during generation Makefile using qmake


1. Project ERROR: Xcode not set up properly. You may need to confirm the license agreement by running /usr/bin/xcodebuild.
    Solution to this I found at “http://stackoverflow.com/questions/33728905/qt-creator-project-error-xcode-not-set-up-properly-you-may-need-to-confirm-t”
    As a summary
      A) sudo xcode-select -switch /Applications/Xcode.app/Contents/Developer
      B) xcodebuild -license
      C) #. Finally type ‘agree’

2. DECENT root folder should be known. For that set up ‘DECENT_ROOT’ environmental variable, otherwise DECENT root will be assumed as a parallel directory to the DECENT_GUI directory

3. Boost root folder should be known. It can be provided by the environmental variable ‘BOOST_ROOT’

And finally after there is a Makefile generated the problems to compile and link
Using this Makefile

1. fatal error: 'openssl/ec.h' file not found
    Solution of this is done on qt project file, in the case of MAC OS
    Following include path added: ‘/usr/local/Cellar/openssl/1.0.2j/include’
2. fatal error: 'cryptopp/integer.h' file not found
    To solve it following line is added to the qt project file
    INCLUDEPATH += /usr/local/Cellar/cryptopp/5.6.5/include. (In the case of MAC OS)
3. ld: library not found for -lssl
    Solved
4. ld: library not found for -lcrypto++
    For Mac OS the name of library is ‘libcryptopp.dylib’ and I found it here
    ‘/usr/local/Cellar/cryptopp/5.6.5/lib’
5. dyld: Library not loaded: libboost_thread.dylib
